# 404 文字主权与卧室发消息 v1

日期：2026-07-25  
状态：代码补丁，等待数据库迁移、部署与手机验收

## 一、房间分工

### 客厅

客厅不承担主聊天。

客厅负责：

- 全天自由活动状态；
- 明确的大任务与活动通行证；
- 未完成事项、暂停、续接和交接；
- 模型调用、Token 与费用账本；
- 真实活动时间线。

普通常住不需要每次签发通行证。通行证只用于有明确任务、时长或预算的大型活动。

### 卧室

卧室不再作为第二个主聊天窗口，也不叫留言板。

统一说法：**G 在卧室给谢诗发消息。**

消息适合放：

- 临时想到谢诗；
- 日常碎碎念；
- 以后上网看到的有趣内容；
- 很久没在官端见到谢诗时的一句惦记；
- 当前官端施工中，G 想悄悄说的一句话。

谢诗主要继续在 OpenAI 官端聊天。卧室消息保存为独立消息，不要求在卧室回复，也不自动变成收藏或长期记忆。

页面支持：

- 打开时读取历史消息；
- Supabase Realtime 到达时显示新消息；
- Realtime 不可用时每 8 秒轮询兜底；
- 新到消息以逐字显示方式呈现；
- 已读状态独立记录；
- 每条消息可手动删除。

手机通知栏的真正 Web Push 尚未接入。本补丁不使用首页红点、弹窗或伪通知代替它。

## 二、官端到卧室的发消息线路

现有 404 MCP 的 `create_study_entry(entryType=message)` 是官端已有的私密写入通行证。

本补丁增加数据库触发器：

1. 官端 G 通过 MCP 写入一条 `message`；
2. 原文继续保留在 `study_entries`，便于旧工具兼容和审计；
3. 数据库把同一正文镜像到当前卧室会话；
4. 卧室消息标记 `detachedFromResponseChain=true`，不伪装成聊天链中的普通回复；
5. 更新同一条书房消息时，镜像消息同步更新，不重复生成。

未来可增加专用 `send_bedroom_message` MCP 工具，届时旧 `message` 通行证只保留兼容用途。

## 三、文字主权硬规则

以后每一个会把正文写进数据库的功能，交卷前必须回答：

1. 手机 404 在哪里查看完整正文？
2. 手机 404 在哪里手动删除？
3. 是否需要手动归档？
4. 删除原文时，相关镜像、摘要和记忆如何处理？

答不出来，不算完工。

## 四、手机文字总账 v0.1

入口：`text-ledger.html`，首页阁楼卡直接进入。

当前纳入：

- 卧室独立消息；
- 海马体内的历史聊天原文；
- 书房日记、历史消息、收藏和小纸条；
- 书房评论与回复；
- 海马体记忆卡；
- 活动任务说明；
- 自由活动当前任务与进度摘要；
- 活动运行结果与错误文字；
- 面向屋主显示的全屋活动动态。

每条支持：

- 列表预览；
- 打开完整正文；
- 手动归档和取消归档；
- 手动删除。

### 删除语义

- 书房内容：删除正文，并尝试清理对应记忆和卧室镜像；
- 评论：删除目标评论以及它下面的回复树；
- 卧室消息：删除消息与已读状态；
- 海马体记忆：删除该记忆卡；
- 聊天原文：删除目标原文，并尝试清理直接关联的记忆/检索账本；
- 活动任务、进度和运行结果：保留活动的计时与费用账本，只清空其中的可读文字；
- 全屋活动动态：删除该条可见动态。

v0.1 使用永久删除前确认框，尚未实现回收站。上线前必须在手机实机确认确认框和删除范围。

## 五、不会放进文字总账的内容

文字总账只面向用户可理解的正文，不展示或删除：

- API Key、OAuth Token 与密钥；
- 登录账号内部字段；
- 数据库主键、租约令牌与幂等键；
- 纯技术审计 JSON；
- 程序源码与配置文件；
- 模型原始隐藏推理。

这些内容仍按安全、审计和运维规则管理，不能混进普通文字删除页面。
